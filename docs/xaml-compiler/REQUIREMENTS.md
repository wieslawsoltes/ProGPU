# ProGPU XAML Compiler Normative Requirements

Status: living normative requirements for the clean-room compiler.
Initial target: `ProGPU.WinUI`.
Planned profiles: Avalonia, WPF, .NET MAUI, and user-defined dialects.

## 1. Requirement language and evidence

`MUST`, `SHOULD`, and `MAY` are normative. Every requirement has a stable ID and must be linked to one or more automated tests or an explicit future-work entry in `FEATURE_MATRIX.md`.

Maturity is recorded independently at five levels:

1. `Syntax`: lossless parse and recovery.
2. `Schema`: vocabulary/type/member representation.
3. `Semantic`: symbol resolution, validation, and bound model.
4. `Emission`: deterministic executable artifact generation.
5. `Runtime`: behavior and hot-reload validation in the target framework.

Passing a lower level MUST NOT be reported as support for a higher level.

## 2. Clean-room and provenance

- `CR-001` The compiler, parser, schema system, emitters, build integration, and runtime seams MUST be original ProGPU implementations.
- `CR-002` External implementations MAY be inspected only for public contracts, observable behavior, tests, architecture, and measured tradeoffs. Their source structure, names, helper types, control flow, comments, generated tables, and implementation text MUST NOT be copied or translated.
- `CR-003` Normative standards and official framework documentation MUST take priority over implementation-specific behavior.
- `CR-004` Every compatibility deviation MUST have a profile flag, diagnostic, rationale, and paired strict/interop tests.
- `CR-005` The vendored `external/microsoft-ui-xaml` tree MUST be used only as a contract/conformance corpus. ProGPU compiler implementation code MUST NOT be derived from the external XAML compiler sources.
- `CR-006` Before integration, the branch MUST be audited for copied code, attribution markers, accidental vendoring, and foreign implementation fingerprints.

## 3. Source text, lexer, and syntax tree

- `SYN-001` Source text MUST preserve the exact UTF-8/UTF-16-decoded character sequence and expose deterministic line/column and absolute-offset mapping.
- `SYN-002` Lexing MUST be lossless: concatenating all non-EOF token text MUST reproduce the input exactly, including whitespace, comments, processing instructions, CDATA, quote style, and entity spelling.
- `SYN-003` Syntax nodes and tokens MUST be immutable after publication and safe to share across compiler passes and threads.
- `SYN-004` The syntax model MUST distinguish XML syntax from XAML semantic constructs. An XML element containing a dotted name is not semantically classified until the XML-to-XAML conversion pass.
- `SYN-005` Strict and recovering parse modes MUST be separate, testable policies. Recovery MUST produce missing/skipped constructs and diagnostics without inventing semantic validity.
- `SYN-006` Diagnostics MUST retain file, absolute span, line/column span, stable ID, severity, and applicable MS-XAML section.
- `SYN-007` The lexer/parser MUST be single-pass `O(N)` average and worst-case time for source length `N`, excluding bounded Unicode classification, with `O(N)` tree/token storage.
- `SYN-008` The parser MUST honor cancellation in bounded intervals and MUST impose configurable limits for nesting depth, token length, attribute length, and total node count.
- `SYN-009` DTD processing, external entities, and external resource resolution MUST be disabled by default.
- `SYN-010` Syntax nodes MUST expose visitor and rewriter APIs that allow independent analyzers and dialect extensions without downcasting to implementation-only types.
- `SYN-011` Stable syntax identity MUST be deterministic and suitable for incremental comparison; identity MUST not depend on process-random hashes.
- `SYN-012` A future incremental parse API MUST reuse unchanged green nodes. Public contracts MUST not prevent a green/red implementation.
- `SYN-013` Names MUST use the Unicode category rules required by MS-XAML rather than ASCII-only approximations.

## 4. XML-to-XAML information set

- `INF-001` XML syntax, the XAML Information Set, and the XAML Schema Information Set MUST be separate models and passes.
- `INF-002` XML namespace mappings MUST be snapshotted on object and member nodes as required by late binding.
- `INF-003` Object, member, text, retrieved-object, namespace, and parent relationships MUST be explicit.
- `INF-004` XML-to-XAML conversion MUST implement MS-XAML section 8 rules as independently testable transforms.
- `INF-005` Whitespace normalization, `xml:space`, content wrapping, member-element recognition, and markup-extension dispatch MUST not be hidden in the XML lexer.
- `INF-006` Markup extensions MUST have a dedicated recursive syntax tree preserving positional/named arguments, nested extensions, quoting, and escape forms.
- `INF-007` Conversion MUST support strict standards behavior and dialect-supplied interop behavior without forking the core parser.

## 4A. Shared markup-value language service

- `MKP-001` Markup-value lexing and parsing MUST live in the framework-neutral `netstandard2.0` core and MUST be reused by WinUI, WPF, Avalonia, MAUI, user profiles, the formatter, the editor, the playground, and IDE hosts.
- `MKP-002` The markup-value lexer MUST operate on Roslyn `SourceText` plus `TextSpan`; tokens MUST carry absolute source spans and MUST NOT require a substring allocation merely to identify token kind or boundaries.
- `MKP-003` The markup syntax model MUST be immutable, thread-safe, recursive, lossless, and Roslyn-isomorphic. It MUST distinguish extension names, positional arguments, named arguments, separators, whitespace/trivia, quoted/unquoted values, nested extensions, escapes, missing tokens, skipped tokens, and custom dialect nodes.
- `MKP-004` The standard grammar MUST represent `{Extension ...}`, nested extensions, `{}` escapes, quoted values, commas, equals signs, braces, whitespace, entity-decoded attribute values with original source mapping, and diagnostics for incomplete or ambiguous constructs.
- `MKP-005` Standard tokenization and structural parsing MUST be framework agnostic. The parser MUST NOT contain hard-coded `Binding`, `x:Bind`, `StaticResource`, `ThemeResource`, `TemplateBinding`, Avalonia, WPF, or MAUI semantic rules.
- `MKP-006` Profiles and users MUST be able to register versioned token recognizers, value-syntax recognizers, parser rules, binders, validators, format rules, lowering rules, and inverse-edit rules without forking the standard lexer/parser.
- `MKP-007` Custom syntax registrations MUST declare trigger tokens/characters, precedence, associativity where applicable, allowed parse contexts, capability/version identifiers, and conflict policy. Ordering MUST be deterministic; overlapping registrations MUST be rejected or diagnosed.
- `MKP-008` Standard parsing MUST be single-pass `O(M)` time and `O(M)` storage for markup length `M`. Nesting, token count, token length, argument count, and diagnostic count MUST have configurable limits and cancellation checks.
- `MKP-009` The hot tokenization path SHOULD be span-index based, avoid per-character virtual dispatch, avoid regular expressions, pool bounded temporary builders where profitable, and defer materializing token/value strings until requested.
- `MKP-010` Incremental markup parsing MUST accept Roslyn `TextChangeRange` values and reuse unchanged immutable subtrees; a change inside one attribute MUST NOT require parsing unrelated markup values.
- `MKP-011` Markup formatters MUST consume the shared token/tree model and return non-overlapping Roslyn `TextChange` values. Formatting MUST preserve custom nodes and unknown syntax losslessly when no registered rule owns them.
- `MKP-012` Markup syntax generation MUST use factories/builders that validate grammar and produce correctly escaped source. Emitters, code fixes, reverse projection, and the playground MUST NOT construct extension text by ad-hoc concatenation.
- `MKP-013` Dialect binding MUST produce explicit bound markup nodes using canonical Roslyn symbols and source mappings. Unknown extensions and arguments MUST remain recoverable error nodes rather than being discarded.
- `MKP-014` The shared test corpus MUST cover every framework dialect, third-party registrations, nested/custom syntax composition, malformed input, Unicode, fuzzing, allocation/throughput baselines, cancellation, limit enforcement, formatting idempotence, and parse-generate-parse equivalence.
- `MKP-015` Public markup lexer/parser contracts MUST remain usable in source generators without loading Workspaces assemblies; workspace-only editing services MUST be layered in a separate package.
- `MKP-016` The shared parser MUST accept arbitrary validated opening-to-closing delimiter maps for member syntaxes that permit commas or equals inside brackets. Heterogeneous pairs MUST nest with a bounded stack, lossless spans/tokens MUST remain unchanged, unterminated or over-depth pairs MUST be diagnosed, and the default empty-map hot path MUST not allocate. Schema-aware hosts MUST select maps per resolved extension member; a project-wide union MUST NOT reinterpret unrelated values.
- `MKP-017` Schema-aware delimiter selection MUST apply to positional as well as named markup-extension arguments. A compiled-binding profile MUST publish its path delimiters through a neutral policy contract so function-call commas and indexers survive initial infoset projection without the shared parser hard-coding an extension or framework name.
- `MKP-018` A custom root markup language MUST register through one immutable `netstandard2.0` language object. Every registration MUST declare a stable ID, host-contract version, positive implementation version, priority, allowed contexts, associativity, conflict policy, capability flags, and nonempty trigger set. Trigger lookup MUST be indexed rather than dispatching every plugin per character. A successful custom parse MUST project into the canonical immutable markup-extension AST so infoset conversion, schema binding, lowering, and emission remain shared. Higher priority wins deterministically; incompatible equal-priority matches MUST produce `PGXAML1154`, explicitly coalescible structurally equivalent matches MAY coalesce, malformed plugin output MUST be diagnosed, and cancellation MUST propagate. Registered inverse formatting/generation MUST round-trip through the same language before source is accepted. Unknown or unowned custom text MUST remain lossless.

## 5. Roslyn foundation, schema, and type system

- `ROS-001` Roslyn `SourceText`, `TextSpan`, `LinePositionSpan`, `Location`, `Diagnostic`, `DiagnosticDescriptor`, `SyntaxAnnotation`, `ISymbol`, `SymbolKey`, `Compilation`, `Workspace`, `Project`, `Document`, and `AdditionalDocument` concepts MUST be reused directly where their contracts fit. Parallel ProGPU abstractions MUST NOT duplicate them without a documented incompatibility.
- `ROS-002` XAML syntax APIs MUST follow Roslyn conventions for green/red immutability, parent/tree/position identity, spans/full spans, missing tokens, trivia, annotations, visitors, rewriters, equivalence, changed-text parsing, and node/token traversal.
- `ROS-003` ProGPU XAML nodes cannot derive from Roslyn `SyntaxNode` because Roslyn's required `GreenNode` construction surface is internal. The isomorphic boundary and adapters MUST be documented and compatibility-tested.
- `ROS-004` Generated C# MUST be constructed with Roslyn C# `SyntaxFactory`/`SyntaxGenerator`, normalized or formatted with Roslyn services, and validated through a Roslyn `Compilation`. Hand-built C# string concatenation is prohibited for production emitters.
- `ROS-004A` Production emitters and framework plugins MUST NOT use `ParseExpression`, `ParseStatement`, `ParseTypeName`, `ParseMemberDeclaration`, text templates, interpolated C# fragments, or equivalent parse-back shortcuts to create known generated constructs. They MUST construct syntax directly from Roslyn symbols, typed IR, tokens, and `SyntaxFactory`/`SyntaxGenerator`. Parsing is permitted only for explicitly user-authored raw-code extension inputs, which MUST be isolated, diagnosed, semantically validated, and capability-gated.
- `ROS-005` Compiler workspace features MUST operate on Roslyn `Solution` snapshots and use `AdditionalDocument` for XAML source where supported.
- `TYP-001` The XAML schema/type contracts MUST be UI-framework-agnostic. Roslyn symbols are the canonical compile-time type system; framework types may appear only as resolved symbols or neutral profile metadata, never as runtime object instances in core passes.
- `TYP-002` The Roslyn provider MUST resolve types, members, events, constructors, generic arguments, accessibility, nullability, collection/dictionary shapes, attached members, and content properties through `Compilation` symbols.
- `TYP-003` Runtime reflection MUST NOT be required for generated construction, property assignment, event hookup, binding, resources, templates, or hot-reload identity.
- `TYP-004` Schema identity MUST be canonical and stable across a compilation. Assignability MUST be explicit and cycle-safe.
- `TYP-005` Intrinsic `x:` and `xml:` types/directives MUST be schema data, not scattered parser special cases.
- `TYP-006` Schema providers MUST support source symbols, metadata symbols, projected/redirected types, generated types, and synthetic framework types.
- `TYP-007` Framework profiles MUST be pluggable without modifying the core parser, infoset, schema, bound IR, or diagnostic infrastructure.
- `TYP-008` User extensions MUST be able to contribute namespaces, directives, type/member projections, markup extensions, value converters, validation rules, lowering rules, and emitters through versioned contracts.
- `TYP-009` Plugin conflicts and ordering MUST be deterministic and diagnosed.
- `TYP-010` Section 5.4 text syntax MUST be represented as immutable schema data containing fixed values, patterns, whitespace/case policy, converter evidence, and an explicit error state. Member text syntax MUST override value-type text syntax without mutating the type descriptor.
- `TYP-011` Intrinsic and enumeration text MUST be validated during semantic binding under sections 6.2.2.5 and 6.3.2.4 while preserving the original lexical text. Framework conversion fallback MUST be explicit and MUST NOT erase an unavailable or invalid schema converter.
- `TYP-012` Attribute-selected converters MUST resolve through canonical Roslyn symbols and validate accessibility, base/interface contract, construction shape, and target compatibility. Bound nodes and IR MUST carry the selected converter; emitters MUST NOT rediscover attributes or use `TypeDescriptor`/runtime reflection.
- `TYP-013` Framework profiles MUST be able to publish immutable synthetic schema types, members, constructors, markup-extension identity, and return types for vocabulary concepts that have no public constructible CLR representation. Synthetic types MUST participate in normal binding and diagnostics without parser special cases or fabricated runtime reflection types.
- `TYP-014` A canonical schema descriptor MUST have a stable provider-qualified identity even when no CLR declaration exists. `ISymbol` is mandatory for CLR-backed declarations but MUST NOT be fabricated for synthetic types, members, constructor parameters, directives, or projected metadata. Equality, diagnostics, caching, and source maps MUST use the descriptor identity and retain a Roslyn symbol only when one genuinely exists.
- `TYP-015` Synthetic markup-extension members MUST declare their name, value type, read/write shape, positional/default-option behavior, and lowering capability as profile data. Named arguments MUST bind through the same canonical member-reference path and diagnostics as CLR-backed members; an emitter MUST NOT accept unknown arguments or recover them by string lookup.
- `TYP-016` A framework profile MUST be able to declare initialization-text syntax for canonical CLR-backed types whose public projection does not expose a converter attribute. The resulting immutable text-syntax descriptor MUST be framework-neutral, preserve lexical text, unwrap nullable value types, and lower only through registered structured Roslyn syntax factories.
- `TYP-017` A profile MUST be able to project a symbol-less semantic member when the public XAML contract intentionally has no corresponding CLR property, field, event, or method. Such a pseudo member MUST have a provider-qualified identity, value type, member kind, semantic capability, and provenance; the compiler MUST NOT fabricate an `ISymbol` or require the runtime object model to add a fake public member.
- `TYP-018` Parser-owned or deferred content MUST remain a distinct schema and IR operation through validation, lowering, source mapping, and hot reload. It MUST NOT be silently discarded, assigned as an ordinary property, or flattened into its owner construction.
- `TYP-019` A framework emitter that advertises deferred-content support MUST construct the deferred object graph as structured Roslyn syntax behind a typed factory/runtime contract, with a fresh namescope and explicit resource/templated-parent context for each instantiation. Unsupported deferred content MUST produce a source-located capability diagnostic before generated C# compilation.
- `TYP-020` Roslyn-side user compiler extensions MUST register through an immutable versioned host. Each registration MUST declare a stable ID, the supported host-contract version, a positive implementation version, priority, capability flags, and conflict policy. The host MUST snapshot registration metadata, reject null, duplicate, incompatible, or capability-invalid registrations, and invoke only the interface corresponding to a declared capability. Ordering MUST be deterministic by descending priority and stable ID. Lower-priority handlers MUST NOT run after a higher-priority handler succeeds. Incompatible equal-priority handlers MUST be diagnosed; explicitly coalescible handlers MAY share a result only when their structured outputs are Roslyn-equivalent.

## 5A. Attribute metadata and symbol-shape conventions

- `ATT-001` Every framework profile MUST publish a versioned inventory of assembly-, type-, member-, method-, and parameter-level attributes that affect XAML schema, binding, construction, serialization, localization, templates, resources, namescopes, tooling, or compilation.
- `ATT-002` Attribute interpretation MUST use Roslyn `AttributeData`, `ISymbol`, and `TypedConstant`. Generator/compiler code MUST NOT instantiate attributes or use runtime reflection.
- `ATT-003` Framework attributes MUST map to stable neutral semantic identifiers through declarative rules. At minimum the model MUST represent content members, runtime-name aliases, implicit dictionary keys, ambient values, dependencies, constructor arguments, content wrappers, namescope/UID/XML-language aliases, whitespace policy, initialization order, converters/serializers, markup-extension return/set handlers, namespace mappings, binding assignment/data-type inheritance, template content/scope, service-provider requirements, and compilation policy.
- `ATT-004` Attribute records MUST retain the canonical `AttributeData`, declaration symbol, provider ID, extracted semantic value, and inheritance provenance. Tooling MUST be able to explain which rule supplied a result.
- `ATT-005` Attribute usage and inheritance MUST be honored according to the registered rule and target kind. Conflicting rules at equal priority MUST produce a diagnostic; providers MUST be ordered by declared priority and stable provider ID.
- `ATT-006` Content-member discovery MUST support both type-level aliases (WinUI/WPF/MAUI style) and member-level markers (Avalonia style), including inherited declarations.
- `ATT-007` Assembly metadata MUST support XML namespace definition, preferred prefix, compatible namespace, root namespace, resource ID/path, and framework compilation options without hardcoding a single framework's attribute names.
- `ATT-008` Unrecognized attributes MUST remain available as raw Roslyn metadata. An attribute in a profile-declared compiler-semantic namespace MUST produce an actionable unsupported-capability diagnostic rather than being silently ignored.
- `ATT-009` Symbol-shape or duck-typing behavior MUST be opt-in profile data with deterministic precedence and provenance. Supported shapes MUST include public collection/dictionary insertion methods, attached getter/setter pairs, add-child contracts, markup-extension suffixes/base/interface contracts, property-system wrappers/registrations, converter methods, factory methods, and framework binding-assignment seams.
- `ATT-010` Shape matching MUST validate accessibility, static/instance form, arity, parameter/ref kinds, generic constraints, target assignability, return type, ambiguity, and overload selection. A name match alone MUST NOT authorize generation.
- `ATT-011` Attribute rules take precedence over shape inference. Explicit user registration takes precedence only when its declared priority is higher; ties and incompatible evidence MUST be diagnosed.
- `ATT-012` Resolved schema descriptors and bound nodes MUST carry the selected attribute/shape evidence. Emitters MUST consume those descriptors and MUST NOT repeat attribute lookup or duck-typing logic.
- `ATT-013` Profiles MUST separately inventory standard XAML/.NET attributes, WinUI metadata, WPF metadata, Avalonia metadata, MAUI metadata, and profile-specific custom attributes. Adding a new profile MUST NOT require changes to the core parser or emitter.
- `ATT-014` Attribute and shape conformance tests MUST cover source and metadata symbols, inheritance, overrides, malformed values, conflicting providers, overload ambiguity, inaccessible candidates, generic collections, implicit keys, member-level content, attached members, trimming/AOT, and deterministic incremental invalidation.
- `ATT-015` Runtime reflection fallback MAY exist only as an explicitly capability-gated compatibility adapter with a diagnostic and removal plan; generated normal paths remain typed and AOT-safe.
- `ATT-016` Repeatable attributes MUST preserve every distinct semantic value and its Roslyn evidence in deterministic provider/declaration order. Single-valued semantics MUST select by declared provider priority and inheritance distance; incompatible evidence at the winning precedence MUST be diagnosed.
- `ATT-017` Standard compiler usage annotations that affect legal or supportable XAML usage, including `ObsoleteAttribute` and `ExperimentalAttribute`, MUST produce diagnostics located in the XAML source with the declared severity, message, and diagnostic identity retained as structured evidence.
- `ATT-018` Ordering metadata such as `DependsOnAttribute` MUST be validated against the canonical member graph, diagnose missing targets and cycles, and lower through a stable topological order. Independent member order and observable evaluation order MUST otherwise be preserved.
- `ATT-019` An implicit dictionary-key convention not exposed by a framework attribute MUST be declared by the owning profile as a type-to-member mapping. The Roslyn provider MUST validate an accessible writable instance member on the exact configured metadata type before publishing the convention; core binding MUST NOT contain framework type-name checks.
- `ATT-020` Attached-member inference MUST support profile-authorized getter-only collection and dictionary shapes. A candidate MUST be an unambiguous accessible static getter with exactly one receiver parameter, a receiver type compatible with the XAML target, and a return type that satisfies the canonical collection/dictionary insertion contract; absence of a setter is valid only for retrieve-and-populate lowering.
- `ATT-021` Attribute aliases, framework projections, pseudo members, and symbol-shape inference MUST converge on the same neutral member descriptor and precedence system. A lower-priority convention MUST NOT shadow explicit metadata, and incompatible winning evidence MUST be diagnosed rather than selected by discovery order.
- `ATT-022` Framework dialect directives that reuse the XAML language namespace but are not part of the baseline MS-XAML schema (for example WinUI `x:Load` and `x:DeferLoadStrategy`) MUST be declared through a versioned profile vocabulary with allowed locations and neutral semantic identities. The binder MUST reject unknown or misspelled `x:` members; the core intrinsic table MUST NOT be polluted with one framework's dialect.
- `ATT-023` Attached-member shape resolution MUST support the framework-authorized self-owner shorthand in which an attached owner qualifier is omitted on an instance of that owner. Ordinary instance members retain precedence; only a fully validated attached getter/setter shape may satisfy the fallback.
- `ATT-024` CLR markup-extension identity MUST require a profile-registered base type, interface, or suffix together with a validated instance `ProvideValue` callable shape; a suffix alone is insufficient. `MarkupExtensionReturnTypeAttribute` (or a registered equivalent) MUST resolve its `Type` value as a Roslyn `ITypeSymbol`, remain attached to the type descriptor, and produce a XAML-located diagnostic when the declared result is not assignable to the target member. Profiles MUST declare base/interface/callable names as data so WPF, WinUI, Avalonia, MAUI, and user-defined duck-typed contracts do not enter the neutral binder as framework-name checks.
- `ATT-025` A CLR markup-extension descriptor MUST retain identity kind, exact base/interface identity symbol when present, selected `IMethodSymbol`, all applicable callable candidates, service-provider parameter type, provider ID, and invalid/ambiguous evidence. A registered one-parameter service-context overload takes precedence over a parameterless overload; among multiple service overloads the unique most-specific parameter type wins by Roslyn assignability. Incomparable interfaces, callable names, or parameter types MUST be diagnosed rather than resolved by declaration or discovery order. Generic registered interfaces MUST dominate their registered non-generic base when the inheritance relation proves that they are more specific.
- `ATT-026` Markup-extension service annotations MUST be canonical schema evidence. `RequireServiceAttribute` (or a registered equivalent) MUST preserve its complete `Type[]` value as Roslyn `TypedConstant` data and publish distinct required `ITypeSymbol` values on the extension descriptor. A profile MUST declare the services its generated/runtime invocation contract can supply and MAY require every extension in that family to declare either required services or acceptance of an empty provider. Missing services, a mandatory-but-absent declaration, and contradictory require/accept-empty evidence MUST invalidate the shape with a XAML-located diagnostic; emitters MUST NOT rediscover or silently ignore this evidence.
- `ATT-027` Object-writer interception metadata such as `XamlSetMarkupExtensionAttribute` and `XamlSetTypeConverterAttribute` MUST resolve the inherited handler-name value to an exact Roslyn callback descriptor. The metadata provider that maps the annotation semantic MUST declare that family's event-args type; an unrelated higher-priority provider MUST NOT replace it through a global map. The selected callback MUST be the unique nearest static, non-generic ordinary method returning `void` with exactly `(object, TEventArgs)` non-`ref` parameters. The descriptor MUST retain the annotation, event-args symbol, selected `IMethodSymbol`, every named candidate, provider provenance, invalid evidence, and whether the current generated assembly can call it directly. Private/inaccessible callbacks are valid public-contract shapes and MUST be marked as requiring a typed profile access bridge; they MUST NOT be mislabeled as malformed metadata or invoked through an undeclared reflection fallback. Missing names/types/methods, instance or incompatible signatures, and equal-precedence conflicts MUST produce XAML-located diagnostics.
- `ATT-028` A symbol-shape provider MUST explicitly declare every feature family it contributes. Registration of one family MUST NOT replace undeclared framework or user families. Independent list contributions MUST compose in deterministic priority/provider order; keyed-map and scalar families MAY be replaced only by an explicitly declared higher-priority contribution. Equivalent winning declarations coalesce. Incompatible equal-priority scalar or same-key declarations MUST become canonical conflict descriptors retaining feature, optional key, every provider ID/priority, and canonical diagnostic values; the binder MUST report each configuration conflict deterministically without allowing provider-ID order to legitimize the selected recovery value. Markup-extension identity, callable names, service-provider types, available services, and require-or-empty policy form a provider-scoped contract: after a base/interface/suffix identity wins, validation MUST use that identity provider's policy rather than an unrelated globally merged policy. Equivalent equal-priority identity contracts MAY coalesce only while retaining every provider ID; incompatible equal-priority identity providers MUST invalidate the shape with all competing provider IDs retained.
- `ATT-029` Load-path text conversion and save-path value serialization MUST remain separate schema capabilities. `TypeConverterAttribute` MUST resolve to a public non-abstract converter, exact public parameterless constructor, and exact public `ConvertFromInvariantString(string)` Roslyn method; bound text and IR MUST retain that descriptor and structured emission MUST call its symbols rather than reconstructing method names. `ValueSerializerAttribute` MUST NOT replace or alter load-path text syntax. Its provider MUST declare serializer base, context, capability-callable, and conversion-callable families; the canonical type/member descriptor MUST retain the annotation, serializer type, constructor, exact `CanConvertToString(object, context)` and `ConvertToString(object, context)` methods, all named candidates, context type, provider, explicit null suppression, and invalid evidence. Save-path syntax generation MUST cast arguments to the selected parameter symbols and MUST NOT instantiate serializers through reflection or C# parse-back.
- `ATT-030` `TrimSurroundingWhitespaceAttribute`, `WhitespaceSignificantCollectionAttribute`, and `UsableDuringInitializationAttribute` MUST project into canonical, provider-qualified boolean schema descriptors retaining the exact `AttributeData`, declaring symbol, typed constant, provider priority/ID, and inheritance depth. Presence attributes MUST resolve to `true`; `UsableDuringInitialization(false)` MUST remain explicit override evidence and MUST suppress an inherited `true`. The schema-neutral infoset MUST retain whitespace text and spans for editing. After type resolution, element-content binding MUST apply default XAML whitespace normalization, inherited `xml:space` scope with `default` reset, whitespace-significant collection retention, and trimming on both sides of annotated child objects. Lexical-only indentation MUST NOT participate in duplicate-member or initial-construction-directive ordering rules. Bound text MUST retain both original and normalized values. Initialization metadata MUST remain canonical ordering evidence until a framework object writer explicitly consumes its top-down attach-before-populate policy; emitters MUST NOT infer such behavior from a type name.
- `ATT-031` Repeatable inherited `ContentWrapperAttribute` metadata MUST become canonical wrapper descriptors retaining the exact annotation/provider/inheritance evidence, wrapper `INamedTypeSymbol`, public parameterless constructor, wrapper content member, and content value type. A valid wrapper MUST be a public non-abstract class assignable to the annotated collection's exact insertion item type and MUST expose a writable or mutable content member. Directly assignable collection content MUST never be wrapped. Foreign content MUST select one applicable wrapper by exact content type and then unique most-specific assignability; no-match, malformed metadata, and equal-best ambiguity MUST be distinct XAML-located diagnostics. Wrapper synthesis MUST create ordinary bound objects/members with derived stable identities, survive IR lowering, and emit through the normal structured Roslyn construction path without reflection, source-text templates, or parse-back.
- `ATT-032` `ConstructorArgumentAttribute` MUST remain save-path serialization and bidirectional-editing metadata; it MUST NOT change ordinary load-path property assignment. A canonical member descriptor MUST retain the exact annotation/provider, argument name, public one-argument constructor, matching `IParameterSymbol`, complete named candidate set, and invalid evidence. The attributed property MUST have public get/set accessors, the argument name MUST be non-empty and ordinally match the parameter, and property/parameter types MUST be symbol-equal. Writers MUST be able to construct the parameterized representation through a structured Roslyn factory that casts to the exact selected parameter type. When that representation is selected, the mapped property is constructor evidence rather than a duplicate serialized member.
- `ATT-033` `AmbientAttribute` and registered equivalents MUST project into canonical type/member Boolean descriptors retaining exact Roslyn annotation, provider, declaring symbol, and inheritance evidence. Effective member ambience MUST be true when either the member/attached getter is annotated or its resolved value type is ambient; attached-property metadata MUST be read from the getter contract and MUST coexist deterministically with setter annotations. Ambient evidence MUST be available to load, save, converter, markup-extension, resource, deferred-content, and editor context services without reflection or framework-name tests. The bound model MUST expose an immutable ambient context graph keyed by stable node/member identity, with nearest-scope-first typed values, type-versus-member origin, owner/member identity, canonical construction ordering, and every crossed deferred boundary. All ambient members on an owner MUST be visible throughout that owner's nested schema scope, including out-of-order type-qualification scenarios. As an immediately executable neutral rule, an effective ambient member whose value type is a canonical dictionary MUST establish a lexical resource scope unless an explicit profile resource role takes precedence. Non-dictionary ambient values MUST remain context-flow evidence and MUST NOT be reclassified as resources.
- `ATT-034` `XamlDeferLoadAttribute` and registered equivalents MUST produce a canonical type/member deferring-loader descriptor from either the two-`Type` or two-string public constructor. It MUST retain the exact annotation/provider, resolved loader/content symbols and declared names, public parameterless loader constructor, exact public instance `Load`/`Save` methods, all named candidates, and malformed/ambiguous evidence. A valid clean-room duck shape MUST pair `Load(reader, IServiceProvider)` with `Save(content, IServiceProvider)` through a symbol-equal reader/return type, accept the declared content type on save, and return either that content type or `object` on load. Member metadata MUST override value-type metadata; either form MUST classify the neutral member as deferred content, permit its node stream without ordinary property assignability or singleton restrictions, lower to `SetDeferredContent`, and establish a deferred ambient-context boundary. Framework profiles MUST consume the descriptor's exact symbols through a typed invocation/factory seam. Structured load/save syntax MUST cast to selected parameter/result symbols and MUST NOT use reflection, C# text templates, or parse-back.
- `ATT-035` Repeatable non-inherited `MarkupExtensionBracketCharactersAttribute` metadata and registered equivalents MUST become canonical member delimiter-pair descriptors retaining exact annotation/provider and both `Char` constants. Identical pairs MUST coalesce by complete attribute identity; one opening character mapped to incompatible closers, malformed constants, whitespace, and reserved markup grammar characters MUST remain explicit invalid evidence with XAML-located diagnostics. Schema consumers MUST derive bounded shared-parser options directly from canonical `XamlMemberInfo`; framework-specific parsers MUST NOT fork the tokenizer.
- `ATT-036` `NameScopePropertyAttribute`, `XmlLangPropertyAttribute`, `UidPropertyAttribute`, and registered equivalents MUST become canonical aliased-member descriptors retaining the exact annotation/provider/inheritance evidence, declared name, direct `IPropertySymbol`, and—where the public contract permits it—exact attached owner `ITypeSymbol` and public static getter/setter symbols. The namescope two-argument `(String, Type)` form MUST resolve against the declared attached owner and target-type applicability. `xml:lang` and `x:Uid` aliases MUST resolve to public writable instance properties. Missing, ambiguous, unreadable, unwritable, malformed-owner, inapplicable-accessor, and getter/setter-type mismatches MUST remain explicit XAML-located diagnostics. A namescope storage property MUST NOT by itself classify every attributed object as a namescope owner; ownership is a separate framework-pluggable type-system shape (System.Xaml defaults it from `INameScope` assignability).

## 5C. Whole-object serialization planning

- `SER-001` Every bound document MUST expose an immutable `XamlSerializationPlanGraph` keyed by stable object IDs. Each object plan MUST retain the bound object/type, ordered member plans, selected alternative constructor if any, and typed issues. Building the default graph MUST be deterministic `O(N + E)` time and `O(N + E)` storage for `N` bound members/objects and `E` dependency edges.
- `SER-002` Each member plan MUST retain the bound member/value nodes, source and serialization indices, canonical member descriptor, effective member-over-value-type serializer, default evidence, conditional/reset methods, optional constructor mapping, and one disposition: omit, directive, attribute, element, content, or constructor argument.
- `SER-003` Save ordering MUST reuse `XamlBoundMemberOrdering`; independent members preserve source order and valid `DependsOn` edges apply the same stable topological order used by semantic contexts and load lowering. Writers MUST NOT implement a second dependency sorter.
- `SER-004` Hidden metadata MUST produce `Omit`; content metadata MUST produce `Content`; explicit attribute preference MUST produce `Attribute`; ordinary supplied attributes/elements MUST retain their source form unless a stronger canonical policy applies. Conditional/default evidence MUST remain runtime-evaluable symbol data and MUST NOT be executed or guessed during compilation.
- `SER-005` Constructor-argument representation MUST be explicitly selected through `XamlConstructorRepresentationMode`; the default MUST preserve ordinary member assignments. Opt-in selection requires exactly one supplied valid mapped member with exactly one value. Multiple alternatives and invalid cardinality MUST produce typed issues without changing the bound document or load IR.
- `SER-006` Structured writer factories MUST consume plan symbols to create Roslyn `ObjectCreationExpressionSyntax` and invocation syntax. Direct calls are permitted only for accessible methods; inaccessible methods require a typed profile/runtime bridge. Generated C# strings, parse-back, reflection, and compilation-time user-code execution are forbidden.
- `SER-007` Conservative reverse projection MAY consume the bound plan. When it does, a literal edit is reversible only if the same stable member has one canonical attribute disposition; policy changes to hidden/content/element/constructor or ambiguous plan identity MUST return a transactional structured conflict and leave XAML unchanged.
- `SER-008` The default plan graph is semantic planning, not a claim of a complete runtime object serializer. Runtime value enumeration, default comparison, conditional invocation, collection/dictionary traversal, serializer contexts, inaccessible-call bridges, namespace/prefix allocation, and lossless text emission MUST be separate typed services layered on this plan.

## 6. Binding, validation, and IR

- `SEM-001` Structural validation and semantic validation MUST be separate ordered passes.
- `SEM-002` Every MS-XAML section 5 and 6 constraint MUST have a stable executable rule ID and coverage entry.
- `SEM-003` The bound tree MUST reference canonical schema symbols and retain source mappings.
- `SEM-004` Unresolved symbols in recovery mode MUST use explicit error symbols; `null` MUST NOT silently mean multiple failure modes.
- `SEM-005` Name scopes, template scopes, resource scopes, dictionary keys, and forward-reference rules MUST be explicit graphs.
- `SEM-006` Lowering MUST produce framework-neutral construction IR with explicit operations for create, retrieve, set, attach, add, dictionary add, invoke extension, enter/exit namescope, register/resolve name, resource lookup, binding, template factory, deferred content, and event hookup.
- `SEM-007` Framework-specific lowering MUST be implemented as passes over bound core constructs, not framework conditionals embedded throughout the parser.
- `SEM-008` Optimization passes MUST preserve diagnostics, source maps, evaluation order, resource order, and namescope semantics.
- `SEM-009` Resource analysis MUST publish immutable scope, definition, and reference records with deterministic identities and source spans. Static and theme references MUST remain distinct; local lexical resolution and forward-reference policy MUST be decided before framework emission and retained on typed IR.
- `SEM-010` A read-only member whose getter returns the explicit object supplied by a property element MUST lower as retrieve-and-populate, not construct-and-assign. Collection and dictionary additions within that retrieved object MUST target the getter result, preserve source order, and never require a nonexistent setter.
- `SEM-011` Resource definitions MUST retain key kind, value type, containing scope, source order, and provider document identity. Cross-file composition MUST invalidate only consumers whose resolved provider set or precedence changed; unresolved references that may be supplied by a framework/application provider MUST remain explicitly external rather than being silently treated as locally resolved.
- `SEM-012` Every resource import MUST retain its semantic URI, stable syntax identity, exact source-value span, and line span in the compilation-independent manifest. Resolution MUST return an explicit resolved, missing, ambiguous, or cyclic outcome; it MUST NOT select an arbitrary suffix match or silently truncate a cycle.
- `SEM-013` A keyed resource-reference object MUST remain both a resource definition and a resource-reference value. Lowering MUST NOT discard it by filtering dictionary values to ordinary object nodes. Reorder-capable analysis MUST retain the exact matched forward definition, alias chains MUST retain a deterministic terminal key for framework platform-resource typing, and cycles or incompatible variant leaves MUST remain explicit diagnostics/evidence.
- `SEM-014` A visual-state runtime MUST resolve state groups from the control-template root scope, stop and restore the prior state's property effects, and apply the requested state's setters and storyboard values through registered typed framework properties without reflection. State changes MUST retain raw dynamic-resource markers and original local-value precedence, capture one restoration value per target/property, and roll back transactionally when assignment fails. Final-value application, duration-based clocks, transitions, nested property paths, and framework-specialized animations MUST be reported as separate capabilities.
- `SEM-015` An ordinary framework binding markup operation MUST remain distinct from compiled binding. Lowering MUST publish the canonical target member, constructed binding descriptor, templated-parent/data context, and lexical namescope lookup root through an optional profile-owned structured Roslyn statement. A profile lacking that capability MUST diagnose the operation; it MUST NOT serialize a binding as an ordinary target value or emit C# text.
- `SEM-016` A framework binding runtime MUST resolve dependency-property path segments through registered identifiers and CLR path segments through generated or explicitly registered typed accessors, without runtime member reflection. Source selection, nested-path rewiring, `OneTime`, `OneWay`, `TwoWay`, `Default`, `PropertyChanged`, `LostFocus`, and `Explicit` updates, converter forward/back conversion, fallback, target-null substitution, element-name lookup, templated parent, replacement, clearing, and template-instance disposal MUST have explicit testable behavior. Notification dispatch MUST remain correct when subscriptions remove or replace themselves.
- `SEM-017` Compiled binding MUST be a distinct schema role, bound value, and construction-IR value. A class-backed binding path MUST resolve from the canonical `x:Class` `INamedTypeSymbol`; every supported segment MUST retain its exact property or field symbol, source type, result type, and write capability. Invalid, ambiguous, unsupported, or non-writable TwoWay paths MUST produce XAML-located diagnostics and MUST NOT degrade to ordinary `Binding`, a runtime path string, registry lookup, or reflection. A profile MUST declare the default mode and any inherited default-mode directive.
- `SEM-018` The shared compiled-binding path reader MUST expose immutable lossless tokens, relative spans, typed syntax steps, typed function arguments, and recoverable error spans on `netstandard2.0`. Member access, constant integer and quoted-string indexers, grouping parentheses, C#-style and pathless casts, parenthesized qualified members, terminal instance/static function calls, path arguments, caret-escaped string arguments, finite integer/floating-point arguments, and language Boolean arguments form the implemented grammar slice. Grouped prefix casts MUST flatten into runtime evaluation order while preserving their original tokens. Event targets and null-propagation MUST extend this syntax family rather than introduce framework parsers or ad-hoc string splitting.
- `SEM-019` A profile-owned `x:DataType` or equivalent directive MUST establish a lexical compiled-binding source-type scope using canonical Roslyn type-name resolution. Bound and IR compiled bindings MUST distinguish root-object and deferred-context sources, retain the exact scoped `INamedTypeSymbol`, and reject an unresolved or unusable type with a XAML-located diagnostic. Structured deferred-factory emission MUST pass the runtime context through the exact typed segment path without reflection, registry lookup, or generated C# parse-back.
- `SEM-020` Compiled-binding casts MUST resolve their target through the lexical XAML namespace map and retain an exact Roslyn target type plus an existing language conversion. A qualified `value.(Owner.Member)` step MUST first resolve a canonical schema attachable getter applicable to the current source type, retaining its optional setter and property-system identity. Only when no attachable member exists may it lower as the documented legacy cast/member form, and that form MUST retain both the exact conversion and readable instance member. Invalid, inaccessible, ambiguous, or non-convertible forms MUST produce `PGXAML2122`.
- `SEM-021` A compiled-binding property function MUST be terminal and bind to one exact accessible, non-generic, non-void `IMethodSymbol`. Instance owner paths and static XAML-qualified owner types MUST remain distinct. Every path argument MUST bind independently from the compiled-binding root and retain its exact symbol path; string, finite numeric, and language-namespace Boolean literals MUST retain their typed value and selected `IParameterSymbol`. Selection MUST reject missing, ambiguous, by-reference, wrong-arity, or implicitly incompatible candidates with `PGXAML2123`. TwoWay function binding MUST require a valid exact BindBack method.
- `SEM-022` A class-backed generated unit containing any compiled property or event binding MUST expose one public `Bindings` lifecycle property with `Initialize`, `Update`, and `StopTracking`. Lifecycle generation MUST be requested through an optional framework profile contract returning structured member and statement syntax. The compiler core MUST own aggregation and ordering; it MUST NOT hard-code a framework lifecycle type or generate C# text. A user-declared `Bindings` member MUST produce `PGXAML3045` and MUST NOT cause a duplicate generated declaration.
- `SEM-023` Every deferred-factory invocation containing a compiled property or event binding MUST own a distinct binding lifetime even when multiple invocations receive the same context object. The compiler core MUST aggregate bindings within that factory boundary and pass a profile-provided registration-owner expression to every property-binding operation. A framework lacking structured deferred lifecycle preparation or finalization MUST produce `PGXAML3046`; the compiler MUST NOT group instances by data item, activate them immediately, or silently omit disposal.
- `SEM-024` Every deferred-factory invocation containing an operation-valued markup extension that requests lifetime ownership MUST receive one profile-created registration owner independent from its source or context object. The assignment profile MUST explicitly report whether it consumed that owner. The compiler core MUST aggregate this evidence, order preparation before construction and finalization after construction, and produce `PGXAML3047` when the selected profile cannot provide or rejects the required lifecycle. This contract MUST remain framework-neutral and extensible to user markup extensions; the core MUST NOT recognize `Binding` or any framework runtime type by name.
- `SEM-025` Ordinary binding MUST have a distinct schema expression role, bound value, and construction-IR value without becoming compiled binding. When a canonical lexical or explicit source type is available, the bound value MUST retain that exact Roslyn type, the lossless shared path syntax, and one exact readable property-or-field symbol, source type, value type, and write capability for every supported CLR path step. Accessor publication is semantic evidence for the ordinary runtime path; it MUST NOT replace the binding descriptor, adopt compiled-binding execution semantics, infer inaccessible members, or use runtime member reflection. Unsupported typed grammar and missing or ambiguous members MUST produce XAML-located diagnostics.
- `SEM-026` Ordinary binding MUST retain an explicit neutral source kind independently from its optional source type. Lexical `x:DataType`, an explicit literal value, an exact public static field/property result, an exact resource-graph result, `RelativeSource Self`, a namescope-resolved `ElementName`, and a profile-resolved deferred-context source MUST remain distinguishable evidence. `Source`, `RelativeSource`, or non-empty `ElementName` MUST suppress lexical data-type inference whenever that selector controls the runtime source; multiple selectors or selectors whose type is not yet provable MUST produce unknown source evidence and MUST NOT publish accessors for the lexical type. Exact static source members, exact resource values, relative-self target types, resolved named-object types, and resolved deferred-context types MUST be canonical Roslyn symbols/types; compiler inference MUST NOT execute a source, resolve a name through reflection, or guess an external element-name or framework context type.
- `SEM-027` A typed ordinary-binding path MUST reuse the shared lossless binding-path syntax and retain one canonical `XamlBindingPathAccessor` per executable member or constant indexer step. Integer indexers MUST resolve exactly one accessible mutable `IList<T>` contract property and string indexers exactly one accessible `IDictionary<string,T>` contract property; the accessor MUST retain the exact indexer symbol, current source type, result type, constant key, and write capability. Unsupported, read-only, or ambiguous indexer contracts MUST produce `PGXAML2132`; the binder MUST NOT infer an indexer from runtime reflection, dynamic access, or collection enumeration.
- `SEM-028` Ordinary `ElementName` source inference MUST reuse the canonical XAML namescope graph after all names in that scope are declared, so backward and forward references have identical exact type evidence. Deferred/template and metadata-declared namescope boundaries MUST isolate names. A name outside the current compiled scope or not represented by a typed XAML object MUST remain runtime-resolved unknown evidence without borrowing a lexical data type; a missing compile-time match alone MUST NOT reject application/runtime name registration.
- `SEM-029` A framework MAY classify an ordinary-binding selector as the source owned by a deferred-content boundary and supply that boundary's exact context type through `IXamlDeferredContentContextTypePolicy`. The neutral binder MUST isolate candidates at every deferred/name-scope boundary, rewrite only bindings classified by the selected profile, and retain the exact context type plus neutral source kind. The compiler core MUST NOT recognize a framework template type, target-type member, relative-source mode string, or runtime class name.
- `SEM-030` Resource type evidence MUST be derived once from the canonical resource graph without constructing a value or invoking a markup extension. Local definitions, resource aliases, conditional variants, and external semantic-manifest candidates MAY provide an ordinary `Binding.Source` type only when every reachable candidate resolves to the same canonical Roslyn symbol. Unresolved, cyclic, forward-disallowed, untyped, or conflicting candidates MUST remain unknown. Enrichment MUST immutably rewrite the bound document after graph construction, preserve stable IDs and the runtime resource descriptor, and expose a neutral `ExplicitResource` source kind without framework names in core analysis.

## 7A. Typed resource keys and project resource graphs

- `RES-001` Resource keys MUST be represented as immutable typed descriptors. At minimum the descriptor MUST distinguish text, type-valued, and public static-member-valued keys and retain source span, stable identity, and canonical Roslyn symbol evidence when binding has occurred.
- `RES-002` `x:Key`, an attribute-selected dictionary-key member, and a profile-declared implicit dictionary-key member MUST flow through the same key factory, duplicate validation, lexical graph, construction IR, emitter, and runtime lookup contracts.
- `RES-003` `x:Type` keys MUST emit `TypeOfExpressionSyntax`; `x:Static` keys MUST emit a symbol-derived `MemberAccessExpressionSyntax`; text keys MUST be converted against the dictionary's canonical key type. No typed key may be serialized to a C# string or reconstructed by parse-back.
- `RES-004` Resource-reference markup MUST preserve nested typed keys. A `{StaticResource {x:Type T}}` or equivalent registered dialect form MUST resolve against the same canonical identity as a matching explicit or implicit type key and MUST pass the typed key expression to the profile runtime seam.
- `RES-005` Compilation-independent per-document manifests MUST retain key kind and a portable canonical identity. Project dependency slices MUST propagate that identity and semantic binding MUST compare external definitions by identity, not display text.
- `RES-006` Runtime resource dictionaries, static resolvers, dynamic-resource markers, theme-resource brush adapters, cycle detection, and invalidation paths MUST accept object-valued keys without coercing them through `ToString`. Framework palette fallback MAY remain string-specific after dictionary scopes have been searched.
- `RES-007` Duplicate-key diagnostics MUST compare canonical runtime-equality identity where compile-time identity is knowable. Roslyn `const` fields MUST fold by declared CLR type and canonical constant value; text keys MUST fold only after their dictionary key conversion type is known. Static properties and non-constant fields whose runtime values cannot be proven equal MUST remain distinct expression identities rather than being guessed equal.
- `RES-008` Provider-independent manifest resolution of framework XML namespaces MUST not guess CLR types. After combining with the Roslyn `Compilation` and selected profile, the generator and CLI MUST semantically enrich raw manifests with canonical bound identities for presentation-namespace, metadata-projected, and implicit typed keys before constructing dependency slices.
- `RES-009` Runtime dictionaries MUST expose a monotonic generation and one mutation notification for each observable logical transaction. Local entry, `IDictionary`, merged collection, theme collection, nested dictionary, and successful `Source` replacement mutations MUST participate; no-op assignments MUST NOT advance the generation.
- `RES-010` Nested merged/theme change propagation MUST be cycle-safe and MUST retain one visited identity set for the whole notification wave. A cyclic dictionary graph MUST notify each dictionary at most once per originating mutation.
- `RES-011` Resource owners MUST subscribe through typed contracts. Element resources MUST invalidate the owning subtree, application resources MUST notify active framework roots, and retained dynamic-resource values MUST re-resolve on the next value/render access without requiring a theme switch.
- `RES-012` Every compiled-resource import MUST retain both its bound lexical-scope owner and a whitespace-insensitive structural scope identity. Definitions reached through that import MUST be visible only from the importing scope and its descendants; an import owned by a nested keyed, template, element, or non-exported conditional dictionary MUST NOT be exported as an unconditional root-provider import. An import in a provider-root conditional partition MAY be exported only with that partition identity intact.
- `RES-013` Cross-file lookup MUST search external providers by nearest lexical scope before applying reverse merged-dictionary and transitive-provider precedence within that scope. Document order MUST NOT allow an ancestor import to override a nearer element-resource import.
- `RES-014` A folded resource key MUST retain a separate expression identity and its canonical Roslyn symbol. Equality, duplicate checks, manifests, and cross-file resolution use runtime-equality identity; structured emission uses the original text, `TypeOfExpressionSyntax`, or symbol-derived member access. Folding MUST NOT replace emitted key expressions with serialized constants.
- `RES-015` A key proven by Roslyn to be the null constant MUST produce a source-located semantic error before emission. A compiler MUST NOT generate a dictionary insertion that is known to fail because its key is null.
- `RES-016` Logical resource identity MUST preserve normalized scheme, authority/component assembly, and path as separate semantics. At minimum the neutral parser MUST represent authority-bearing `ms-appx` and `avares` forms, WPF `Assembly;component/Path`, WPF `pack://application:,,,` component/local forms, root-relative paths, and document-relative paths without flattening qualified identities into path suffixes.
- `RES-017` A qualified import MUST resolve only in its declared authority or component assembly. Relative and root-relative imports from a qualified provider MUST remain inside that provider authority. Only an explicitly authority-free current-package form MAY use the documented unqualified-project compatibility fallback, and ambiguity MUST still fail deterministically.
- `RES-018` Conditional resources MUST be represented as sibling partition scopes, never flattened into their owning lexical dictionary. Each partition MUST retain a typed key and parent scope; duplicate keys are invalid within one partition but MAY occur in distinct partitions.
- `RES-019` A static or dynamic/theme lookup outside a conditional partition MUST retain every applicable active-theme candidate at the nearest lexical level. Static lookup snapshots the selected candidate at load time; dynamic/theme lookup remains eligible for re-evaluation after a theme change. A reference originating inside one partition MUST search that partition and then its lexical ancestors without leaking into sibling partitions.
- `RES-020` Conditional partition identity MUST survive raw and semantic manifests, provider visibility, transitive `Source` imports, external-definition slices, equality, and fingerprints. A provider-root theme partition is exported conditionally; an ordinary provider imported from inside a theme partition inherits that consumer partition.
- `RES-021` Getter-only collection/dictionary retrieval MUST distinguish an existing property object from a keyed item. A single keyed dictionary child is an item even when its CLR type is assignable to the getter-only property type; it MUST NOT be misclassified as the retrieved property object.
- `RES-022` Conditional external resolution MUST retain provider provenance, definition stable identity, and partition key for every applicable variant. Lookup precedence MAY select a first runtime candidate, but compile-time graph data MUST NOT erase the remaining variants needed for validation, tooling, hot reload, and dependency analysis.
- `RES-023` Runtime theme lookup MUST accept contrast-theme state independently of `ElementTheme`. When contrast mode is active, a `HighContrast` partition MUST take precedence over requested Light or Dark and MUST fall back to `Default` only when the contrast partition is absent. Merged-dictionary traversal MUST preserve the selected partition and remain cycle-safe.
- `RES-024` A framework runtime MUST expose a typed, reflection-free platform-resource provider contract carrying the effective theme, theme family, and contrast state. Provider replacement and provider change notification MUST invalidate cached platform brushes and active theme-resource consumers. Application resources retain precedence; unresolved provider keys continue to the documented built-in platform fallback and unknown keys still fail.
- `RES-025` An untyped resource, setter, visual-state, or template-binding value crossing into a typed framework property MUST pass through one framework-owned XAML value-conversion contract. Initial assignment and dynamic re-evaluation MUST use the same conversion, unwrap nullable target types without changing their public property identity, use invariant culture, preserve already assignable values, and fail explicitly for unsupported conversions. Per-control casts and divergent receiver-specific converters are prohibited.
