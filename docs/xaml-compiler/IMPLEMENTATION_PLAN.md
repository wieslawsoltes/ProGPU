# ProGPU XAML Compiler Implementation Plan

Status: living normative delivery plan. Maturity claims remain governed by `FEATURE_MATRIX.md` and require automated evidence.

## Delivery principles

Every gate extends the same lossless syntax, schema, bound model, construction IR, Roslyn emission, profile, and workspace contracts. Framework features are never implemented as parser shortcuts or emitted C# text. Each gate includes clean-room provenance, negative diagnostics, generated-code compilation, incremental evidence, runtime behavior where applicable, and a feature-matrix update.

## Gates

Provider-registered `ShouldSerialize`/`Reset` duck shapes add exact conditional/reset callables and reject invalid shapes or `DefaultValue` mixing without executing user code. Full object-graph serialization remains later gate work. Fuzz and performance baselines remain part of gate 1.
